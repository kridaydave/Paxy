Architecture Design Document

Paxy CLI

A command-line pixel art maker for AI agents — create, draw, layer, animate, and export pixel art from the terminal.

v1.0 Design Node.js + ESM 15 files ~1200 LOC
TL;DR

Paxy is a single-file HTML pixel art editor (2321 lines). We're converting it into a Node.js CLI tool that AI agents call as a bash command. Agents create canvases, draw pixels/shapes, manage layers and frames, and export to PNG/GIF — all via paxy <subcommand> invocations.

State persists in .paxy JSON files. Every command loads → mutates → saves. No daemon, no REPL, no state between calls.

Requirements

Functional capabilities the CLI must provide, derived from the original paxy editor's feature set — scoped for agent consumption.

Core
Project Management
Create, load, save .paxy JSON files. Resize canvas. Inspect project metadata.
Drawing
Pixel Operations
Set pixels, draw lines, rectangles, circles. Flood fill. Erase. Symmetry modes.
Structure
Layers & Frames
Per-frame layer stacks with opacity, visibility, merge, reorder. Frame duplication for animation.
Color
Palette System
12 built-in palettes. Custom color import from .hex / .gpl files.
Output
Export Formats
PNG (scaled), spritesheet PNG + JSON manifest, animated GIF with per-frame timing.
Extras
Templates
12 built-in pixel art stencils (Heart, Sword, Tree, etc.) placed centered on canvas.

Non-Functional

  • Zero-config: npx paxy or global install, no setup step
  • Agent-friendly: every command prints JSON on success, errors are structured
  • Stateless per invocation: each subcommand loads file → mutates → saves
  • Max canvas 64×64 (pixel art constraint, keeps files under 500KB)

Constraints

Runtime
Node.js
User's choice. ESM modules, "type": "module" in package.json.
Interface
Pure Subcommands
No REPL, no interactive mode. Every operation is a single CLI invocation.
Packaging
Single npm Package
Publishable to npm. bin entry point exposes paxy command.

Architecture

A flat, module-per-concern structure. CLI parsing at the top, shared library modules in the middle, command handlers at the bottom.

.paxy File Format

A JSON-based project format using sparse pixel arrays — only pixels that have been drawn are stored, not entire grids.

Design Choice

A 32×32 layer with 10 drawn pixels stores 10 entries, not 1024. Agents map directly: paxy pixel --x 5 --y 3 --color #ff004d → one JSON object in the array.

{
  "version": 1,
  "canvas": { "width": 32, "height": 32 },
  "palette": ["#000000", "#1d2b53", "#7e2553", ...],
  "activeFrame": 0,
  "activeLayer": 0,
  "frames": [
    {
      "duration": 200,
      "layers": [
        {
          "name": "Layer 1",
          "visible": true,
          "opacity": 1.0,
          "pixels": [
            { "x": 5, "y": 3, "color": "#ff004d" },
            { "x": 6, "y": 3, "color": "#ff004d" }
          ]
        }
      ]
    }
  ]
}

Transparent pixels are simply absent from the array. Rendering composites visible layers bottom-to-top — last pixel at any (x,y) wins.

Command Reference

Every command accepts --project <path> (defaults to ./untitled.paxy). All commands print JSON on success.

Project
paxy create --size 32x32 --output art.paxyCreate empty project
paxy info --project art.paxyPrint canvas size, frame count, layer counts
paxy resize --size 48x48 --project art.paxyResize canvas (preserves pixels)
Draw
paxy pixel --x 5 --y 3 --color #ff004d --project art.paxySet one pixel
paxy line --x1 0 --y1 0 --x2 10 --y2 10 --color #ff004d --project art.paxyBresenham line
paxy rect --x1 2 --y1 2 --x2 8 --y2 8 --color #ff004d --filled --project art.paxyRectangle (outline or filled)
paxy circle --cx 16 --cy 16 --r 5 --color #ff004d --filled --project art.paxyCircle (midpoint algorithm)
paxy fill --x 5 --y 5 --color #ff004d --project art.paxyFlood fill
paxy erase --x 5 --y 3 --project art.paxyErase pixel
paxy symmetry --mode h --project art.paxySet symmetry (off / h / v / hv)
paxy template --name Heart --project art.paxyPlace template centered
Layers
paxy layer add --name "BG" --project art.paxyAdd new layer
paxy layer remove --name "BG" --project art.paxyRemove layer (min 1)
paxy layer rename --from "Layer 1" --to "BG" --project art.paxyRename layer
paxy layer list --project art.paxyList all layers with opacity/visibility
paxy layer active --name "BG" --project art.paxySet active layer
paxy layer opacity --value 0.5 --project art.paxySet active layer opacity
paxy layer visible --name "BG" --show false --project art.paxyToggle visibility
paxy layer merge --project art.paxyMerge active layer down
paxy layer move --direction up --project art.paxyReorder layer
Frames
paxy frame add --project art.paxyAdd empty frame after current
paxy frame duplicate --project art.paxyDeep-clone current frame
paxy frame remove --project art.paxyDelete frame (min 1)
paxy frame list --project art.paxyList frames with duration & layer count
paxy frame active --index 2 --project art.paxySet active frame
paxy frame move --direction left --project art.paxyReorder frame
paxy frame duration --ms 300 --project art.paxySet frame duration (ms)
Palette
paxy palette listList all built-in palettes
paxy palette set --name "Pico-8" --project art.paxySwitch project palette
paxy palette add-color --hex #ff004d --project art.paxyAppend color to palette
paxy palette import --file custom.hex --project art.paxyImport from .hex / .gpl file
Export
paxy export png --scale 4 --output frame.png --project art.paxyExport current frame as PNG
paxy export spritesheet --cols 4 --output sheet.png --project art.paxySpritesheet PNG + JSON manifest
paxy export gif --output anim.gif --project art.paxyAnimated GIF (all frames)

Agent Usage Example

A typical workflow an AI agent would execute — creating a character, adding layers, and exporting.

Agent Workflow
# Create a 32×32 canvas
paxy create --size 32x32 --output hero.paxy

# Draw a character body
paxy pixel --x 15 --y 10 --color #ff004d --project hero.paxy
paxy pixel --x 16 --y 10 --color #ff004d --project hero.paxy
paxy line --x1 15 --y1 11 --x2 15 --y2 18 --color #7e2553 --project hero.paxy
paxy line --x1 16 --y1 11 --x2 16 --y2 18 --color #7e2553 --project hero.paxy
paxy circle --cx 15 --cy 8 --r 2 --color #ffa300 --filled --project hero.paxy

# Add a background layer
paxy layer add --name "BG" --project hero.paxy
paxy layer active --name "BG" --project hero.paxy
paxy fill --x 0 --y 0 --color #1d2b53 --project hero.paxy

# Export for review
paxy export png --scale 4 --output hero.png --project hero.paxy
paxy export gif --output hero-walk.gif --project hero.paxy

Failure Modes

What can go wrong and how each case is handled.

FailureMitigation
Missing project fileClear error: "File not found: art.paxy. Use paxy create first."
Drawing out of boundsClamp to canvas bounds, warn on stderr
Layer index out of rangeError with valid range hint
GIF export with 1 frameError: "Need 2+ frames for GIF"
Corrupt .paxy JSONValidate on load, clear parse error message
Unknown palette nameList available palettes in error output

Revisit If

  • Canvas sizes >64×64 become needed (sparse format handles it, but export slows)
  • Agents need batch operations (e.g. draw 100 pixels in one call → add paxy batch --file steps.json)
  • Real-time collaboration or merge conflicts matter (out of scope for CLI)
