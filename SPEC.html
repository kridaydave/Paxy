Design & Architecture Spec

Paxy CLI

A command-line pixel art maker for AI agents

v1.0 Design Node.js + ESM 15 files ~1200 LOC
TL;DR

Paxy is a single-file HTML pixel art editor (2321 lines). We’re converting it into a Node.js CLI tool that AI agents call as a bash command. Agents create canvases, draw pixels/shapes, manage layers and frames, and export to PNG — all via paxy <subcommand> invocations.

State persists in .paxy JSON files. Every command loads → mutates → saves. No daemon, no REPL, no state between calls.

Problem

AI image generators can’t produce precise pixel art — correct grid, exact colors, intentional shapes.

AI assistants need a CLI tool to create pixel art with pixel-level control, so humans get accurate sprites they can review as PNG and refine in the GUI.

Users

Primary
AI Assistants
AI assistants (opencode) generating pixel art on behalf of humans via CLI commands.
Secondary
Humans
Reviewing the exported PNG and opening .paxy in the GUI to polish.

Success Metric

Target

An AI agent can create a recognizable character sprite (template + custom pixels) and export a PNG — all via CLI commands — in under 30 seconds of tool calls.

Kill Condition

After building, if you can’t get an AI to successfully generate a recognizable sprite via CLI in <30 seconds of tool calls, the tool failed its primary user.

Architecture

Data Format Decision (One-Way Door)

Use the GUI’s existing dense grid format — not the sparse format. The GUI stores grid[y][x] → null or "#ff004d". Dense means zero conversion layer, automatic GUI compatibility, and files that just work.

PNG Export Decision

pngjs (pure JS) over sharp. For a 64×64 grid at 8x scale (512×512px PNG), pngjs takes ~200ms. Sharp’s 7MB native binary is permanent install pain for zero visible benefit.

Command Reference

Project
paxy create Create a new .paxy project file with given canvas size
paxy info Print project metadata, canvas size, layer/frame count, palette
paxy resize Resize the canvas (pads or clips existing pixel data)
Draw
paxy pixel Set a single pixel at (x, y) with a color
paxy line Draw a line between two points using Bresenham’s algorithm
paxy rect Draw a rectangle (outline or filled) between two corners
paxy circle Draw a circle (outline or filled) using midpoint algorithm
paxy fill Flood fill from a point using stack-based algorithm
paxy erase Set a pixel to null (transparent)
paxy template Place a built-in stencil (Heart, Sword, Tree, etc.) centered on canvas
paxy batch Execute a JSON array of pixel/shape operations in one call
Palette
paxy palette list List all colors in the current palette
paxy palette set Set the active palette by name
paxy palette add-color Add a hex color to the current palette
paxy palette import Import colors from a .hex or .gpl file
Export
paxy export png Export the active layer as a scaled PNG file

Agent Usage Example

Shell Session
# Create a 32×32 canvas
paxy create --size 32x32 --output hero.paxy

# Draw a character body
paxy pixel --x 15 --y 10 --color #ff004d --project hero.paxy
paxy line --x1 15 --y1 11 --x2 15 --y2 18 --color #7e2553 --project hero.paxy
paxy circle --cx 15 --cy 8 --r 2 --color #ffa300 --filled --project hero.paxy

# Place a template
paxy template --name Heart --project hero.paxy

# Batch draw
paxy batch --file character.json --project hero.paxy

# Export for review
paxy export png --scale 4 --output hero.png --project hero.paxy
